# Hackathon Work

Netify existed before HackRome as a local PDF-to-video generation pipeline. The hackathon work focused on turning it into a product people could use and developers could integrate with.

## Built During HackRome

- Integrated ElevenLabs narration so generated lessons have a high-quality teacher voice.
- Connected Supabase Auth for user accounts.
- Connected Supabase Postgres for profiles, uploaded resources, and generated video metadata.
- Connected private Supabase Storage buckets for source PDFs and finished videos.
- Built a local web flow for uploading PDFs, starting generation, watching progress, and returning to generated videos.
- Added a developer API shape for async render jobs and live progress events.
- Documented the API with OpenAPI and example clients.
- Created a Netify explainer PDF designed to generate the Devpost demo video.

## Reliability Work

The pipeline generates executable animated scenes, so reliability work matters as much as initial generation. During the hackathon, render failures were investigated and the local repair path was improved so common generated-code issues can be recovered without exposing implementation details in this public repo.

## Demo Framing

The strongest proof is recursive: the demo video explaining Netify is generated by Netify from the included explainer PDF.

The honest current limitation is deployment. The render backend is not publicly hosted yet; the demo runs locally, with a cloud-ready architecture planned for durable rendering.
